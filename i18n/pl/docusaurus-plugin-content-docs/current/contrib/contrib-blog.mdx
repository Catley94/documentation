# Napisz poradnik

Postępuj zgodnie z tym tutorialem, jeśli chcesz napisać poradnik do Archcrafta...

## Wymagania wstępne

- [Skonfigurowany system](setup-env.mdx) do pracy lokalnej.
- Powinieneś mieć podstawową wiedzę na temat składni [Markdowna](https://www.markdownguide.org/cheat-sheet) i [MDXa](https://mdxjs.com). Zobacz [Docusaurus Markdown Features](https://docusaurus.io/docs/markdown-features).
- Powinieneś przynajmniej wiedzieć jak: *zforkować*, *zklonować* repozytorium i utworzyć *pull requesta*.

## Pliki i formatowanie

Katalog **blog** zawiera wszystkie pliki `.md` i `.mdx` do poradników.

- Aby dodać nowy poradnik, wystarczy utworzyć nowy plik w odpowiednim formacie.
- Format jest tutaj ważny, struktura formatu to :
```
YEAR 	# Katalog
├── month-date-post-NAME.mdx	# plik .md lub .mdx
└── month-date					# Katalog
   ├── Images					# Katalog
   └── post-NAME.mdx			# plik .md lub .mdx
```

## Tytułi tagi

Dodaj odpowiedni slug, tytuł i tagi w `front matter` tworzonego pliku.
```yml
---
slug: NUM-post-NAME
title: TITLE FOR THE POST
authors: ID
tags: [TAG1, TAG2]
---
```

## Autor

Dodaj swoje dane w pliku **`authors.yml`** i dodaj swój identyfikator w `front matter` utworzonego pliku.

`authors.yml`
```yml
...
ID:
  name: YOUR_NAME
  title: ABOUT_YOU
  url: https://github.com/USERNAME
  image_url: https://github.com/USERNAME.png
...
```

`Front matter`
```yml {3}
slug: NUM-post-NAME
title: TITLE FOR THE POST
authors: ID
tags: [TAG1, TAG2]
```

> Aby uzyskać więcej informacji, odwiedź [Docusaurus:Blog](https://docusaurus.io/docs/blog)
